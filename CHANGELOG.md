# 강화중독 패치노트

## 2026-06-01

### Web Forms SPA 초기 골격

- 게임 이름을 `강화중독`으로 정했습니다.
- `Default.aspx`, `.aspx.cs`, `.ashx` API 기반의 새 프로젝트를 만들었습니다.
- 페이지 새로고침 없이 직접 사냥과 강화가 동작하도록 구성했습니다.
- 밸런스 수치와 게임 처리 코드를 분리하고 주요 수정 지점에 한글 주석을 추가했습니다.

### MSSQL 저장 연결

- `enhance_addiction` 전용 로컬 DB 생성 스크립트를 추가했습니다.
- 사용자 상태를 MSSQL `dbo.players` 테이블에 저장하도록 변경했습니다.
- 강화 시도를 `dbo.enhancement_attempts` 테이블에 누적하도록 구성했습니다.
- 게임 상태에 영향을 줄 수 있는 요청을 `dbo.game_action_logs` 테이블에 기록하도록 구성했습니다.
- 감사 로그에는 성공 여부, 메시지, 행동 전후 상태와 상세 정보를 저장합니다.

### 기존 게임 시스템 전체 이식

- 기존 ForgeIdle의 사냥터 12개, 보스 11마리와 +30 강화 확률표를 옮겼습니다.
- 자동 사냥, 직접 사냥, 레벨, 경험치, 보호권, 스탯 시스템을 옮겼습니다.
- 닉네임 설정과 실시간 랭킹을 추가했습니다.
- 자동 사냥, 직접 사냥, 강화, 보스, 스탯, 닉네임 요청을 공통 감사 로그에 연결했습니다.

### 카카오 로그인과 운영 배포

- 카카오 로그인, 로그아웃, 로컬 전용 테스트 로그인 기능을 추가했습니다.
- 운영 서버 환경 변수에서 카카오 비밀값과 MSSQL 연결 문자열을 읽도록 변경했습니다.
- 기존 ForgeIdle 운영 DB와 충돌하지 않는 `ea_` 접두사 테이블을 추가했습니다.
- 서버 최초 실행 시 강화중독 전용 테이블을 자동 생성하도록 구성했습니다.
- GitHub Actions 기반 MonsterASP.NET 자동 배포 워크플로를 추가했습니다.
