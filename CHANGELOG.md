- C# 메소드와 JavaScript 함수 위에 담당 기능을 설명하는 한글 주석을 추가했습니다.
- 이전 데이터 이전 상태를 조회하는 SQL과 검증 후 수동으로 테이블을 정리하는 SQL을 분리했습니다.

## 2026-06-01

### Web Forms SPA 초기 골격

- 게임 이름을 `강화중독`으로 정했습니다.
- `Default.aspx`, `.aspx.cs`, `.ashx` API 기반의 새 프로젝트를 만들었습니다.
- 페이지 새로고침 없이 직접 사냥과 강화가 동작하도록 구성했습니다.
- 밸런스 수치와 게임 처리 코드를 분리하고 주요 수정 지점에 한글 주석을 추가했습니다.

### MSSQL 저장 연결

- `enhance_addiction` 전용 로컬 DB 생성 스크립트를 추가했습니다.
- 사용자 상태를 MSSQL `dbo.players` 테이블에 저장하도록 변경했습니다.
- 강화 시도를 `dbo.enhancement_attempts` 테이블에 누적하도록 구성했습니다.
- 게임 상태에 영향을 줄 수 있는 요청을 `dbo.game_action_logs` 테이블에 기록하도록 구성했습니다.
- 감사 로그에는 성공 여부, 메시지, 행동 전후 상태와 상세 정보를 저장합니다.

### 기존 게임 시스템 전체 이식

- 기존 ForgeIdle의 사냥터 12개, 보스 11마리와 +30 강화 확률표를 옮겼습니다.
- 자동 사냥, 직접 사냥, 레벨, 경험치, 보호권, 스탯 시스템을 옮겼습니다.
- 닉네임 설정과 실시간 랭킹을 추가했습니다.
- 자동 사냥, 직접 사냥, 강화, 보스, 스탯, 닉네임 요청을 공통 감사 로그에 연결했습니다.

### 카카오 로그인과 운영 배포

- 카카오 로그인, 로그아웃, 로컬 전용 테스트 로그인 기능을 추가했습니다.
- 운영 서버 환경 변수에서 카카오 비밀값과 MSSQL 연결 문자열을 읽도록 변경했습니다.
- 기존 ForgeIdle 운영 DB와 충돌하지 않는 `ea_` 접두사 테이블을 추가했습니다.
- 서버 최초 실행 시 강화중독 전용 테이블을 자동 생성하도록 구성했습니다.
- GitHub Actions 기반 MonsterASP.NET 자동 배포 워크플로를 추가했습니다.

## 2026-06-01 (추가 패치)

### 게임 시스템 밸런스 조정
- **직접 사냥 쿨타임 단축**: 몬스터 처치 대기 시간을 3초에서 **1초**로 줄였습니다.
- **직접 사냥 효율 상승**: 쿨타임 단축에 따라 시간당 기대 수익이 자동 사냥 대비 골드 **450%**, 경험치 **375%**로 상향되었습니다.

### UI/UX 개선
- **실시간 보상 트래킹**: 이제 자동 사냥 중 정산 버튼을 누르기 전에도 현재까지 누적된 골드와 경험치를 실시간으로 확인할 수 있습니다.
- **타이머 정밀도 개선**: 서버 전송 시간 포맷을 밀리초 단위로 정밀화하여, 사냥 시작 시 시간이 마이너스로 표시되거나 밀리는 현상을 해결했습니다.

### 기술 수정 사항
- `GameService.cs`: `ManualHuntCooldown` 수치 변경 및 `Iso()` 시간 포맷 수정
- `game.js`: `updateTimer` 내 실시간 보상 계산 로직 추가 및 `Math.max`를 이용한 시간 보정 적용
- `Default.aspx`: 직접 사냥 효율 변경에 따른 안내 텍스트 업데이트

### 이전 사용자 데이터 자동 이전

- 기존 ASP.NET Core 버전의 `dbo.accounts` 사용자 상태를 새 `ea_` 테이블로 자동 이전합니다.
- 기존 강화 이력도 새 통계 테이블로 한 번만 이전합니다.
- 이전 버전의 원본 테이블은 삭제하지 않고 보존합니다.
